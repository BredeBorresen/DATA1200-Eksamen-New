GitHub gruppesamarbeid

Som gruppe var vi to første klassinger og to tredje klassinger. Det ble holdt en grundig opplæring i hvordan GitHub fungerer og det ble tillat rom for læring slik at det ble mer interessant å jobbe med prosjektet. Vi fikk utforsket både med og uten branching, slik at vi kunne lage branch om vi ville eller ikke. Noen ganger lå noen foran og andre bak som kunne bli litt krevende, men med god hjelp så kom vi oss fort tilbake på sporet

Vi brukte også LiveShare i VS Code for code review og gi læring innad i gruppa noe som var til meget god hjelp for et godt samarbeid. Under ser du en oversikt over commit-historikken vår, som viser hvordan prosjektet har utviklet seg over tid:

Fordeler med GitHub

Samarbeid med læring rundt git

Vi møtte på noen utfordringer underveis, men med godt samarbeid kom vi oss gjennom det. Vi hadde gjevne møter hvor de med mest kompetanse innenfor git viste og forklarte hvordan det fungerte. Dette var svært lærerikt og bidro til mye læring. Målet var at alle på gruppa skulle lære å bruke git selvstendig så vi kunne ta nytte av dens egenskaper.

Github

GitHub har revolusjonert måten utviklere samarbeider på ved å tilby en strukturert og effektiv plattform for versjonskontroll og prosjektstyring. I repositorier som GitHub kan flere brukere jobbe samtidig på samme prosjekt, noe som gir en smidig arbeidsflyt og muligheten til å dele kunnskap og ressurser.

Brancher

En av de viktigste fordelene med GitHub er funksjonen for branching. Branching gjør det mulig for teammedlemmer å jobbe på separate funksjoner eller feilrettinger uten å påvirke hovedkoden. Når arbeidet er ferdig, kan det gjennomgås via pull requests før det flettes tilbake til hovedgrenen. Dette sikrer at koden som legges til er testet, gjennomgått og oppfyller kvalitetskravene.

Commits

Commit-historikken spiller også en sentral rolle ved å gi oversikt over alle endringer som er gjort i prosjektet. Hver commit inkluderer informasjon om hva som ble endret, hvem som gjorde det, og når. Dette gjør det enkelt å spore endringer, finne feil og forstå prosjektets utvikling.

Konflikthåndtering

Konflikthåndtering er en annen styrke ved GitHub. Når to eller flere bidrag er i konflikt, identifiserer GitHub dette og gir verktøy for å løse konfliktene. Dette gjør det mulig å opprettholde en strukturert og harmonisk arbeidsflyt.

Alt i alt har vi lært mye gjennom bruk av GitHub som vi skal ta med oss videre.
