GitHub-samarbeid

Som gruppe var vi to første klassinger og to tredje klassinger.

Det ble holdt en grundig opplæring i hvordan GitHub fungerer og det ble tillat rom for læring slik at det ble mer interessant å jobbe med prosjektet.

Vi fikk utforsket både med og uten branching, slik at vi kunne lage branch om vi ville eller ikke.

Noen ganger lå noen foran og andre bak som kunne bli litt krevende, men med god hjelp så kom vi oss fort tilbake på sporet.

Vi brukte også LiveShare i VS Code for code review noe som var til meget god hjelp for et godt samarbeid.

Under ser du en oversikt over commit-historikken vår, som viser hvordan prosjektet har utviklet seg over tid:

Fordeler med GitHub

Fordelene med å bruke GitHub var blant annet at det bidro til et bedre samarbeid, ved at alle kunne se hvilke endringer som ble gjort med commit historikk.

Det var også nyttig, siden det hjalp til med å skape en bedre oversikt, ettersom at konflikter mellom filer ble oppdaget og løst på en strukturert og organisert måte.

På denne måten ble det også enklere å integrere arbeidet fra alle gruppemedlemmene.

GitHub bidro til god kommunikasjon og oversikt over progresjonen til oppgaven.

Alt i alt har vi lært mye gjennom bruk av GitHub som vi skal ta med oss videre.

Vi møtte på noen utfordringer underveis, men med godt samarbeid kom vi oss gjennom det.
